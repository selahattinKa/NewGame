Adversarial design review'larda spesifik sorular belirleyerek yönlendiriyor — o soruların tamamını eksiksiz yanıtla, genel review'a dağılma.

[BLOCKING] / [RECOMMENDED] / [NICE-TO-HAVE] formatında yapılandırılmış output tercih ediyor.

Oyun tasarımı kararlarında karşılaştırmalı örnek oyunları (AFK Arena, Idle Heroes, Monster Warlord, Summoners War) referans almak bekleniyor — soyut teori yeterli değil, "bu oyun nasıl yapıyor?" sorusu da yanıtlanmalı.

Pillar tasarım testlerini kullanarak bulguları doğrulamayı bekliyor — "Pillar X şöyle diyor, bu tasarım onu şöyle ihlal ediyor" yapısı.

**Why:** İlk etkileşimde gözlemlendi.
**How to apply:** Her adversarial bulguyu bir sütuna bağla, genre referansıyla destekle, severity açıkça belirt.
